Excel Dashboards and Reports：Part 1

Introducing Dashboards

Dashboards 和 Reports 的定义

Reports（报告）

Reports 的关键属性在于：它不会引导读者得出预先已经定义好的结论。（即使这个Reports 包含了各种复杂的元素，如钻取、分析、聚合计算、交互式图表等等。）

如下面这个Report，它统计了每年国家公园的游客数，其中可能蕴含了某些信息（或结论），但是，Report 并没有事先给出某些结论，而只是展示了一些统计数字。

Dashboards（仪表板）

Dashboards 则包含了三个关键属性：

  1. 以图形方式显示数据(如图表)。提供可视化，帮助读者关注关键趋势、比较和异常；
  2. 仅显示与仪表板目标相关的数据；
  3. 包含与仪表板的目标相关的预定义结论，使读者不必自己去分析。

如下面这个仪表板：首先，它是一个可视化的视觉信息，能让我们迅速关注到国家公园参观人数的变化趋势。其次，并不是所有的细节数据都在上面显示了，它仅展示了与分析目标相关的数据信息。最后，它能引导我们得出相应的结论。（趋势究竟是如何变化的？）

确定用户需求

在为仪表板收集用户需求时，关注您需要的数据类型、需要的数据维度、将使用的数据源等等，这是一件好事，没有可靠的数据处理，您的仪表板将不会有效或可维护。

明确需求来源

在收到仪表项目需求时，应找到提出需求的人进行沟通，有时可能并不需要复杂的仪表板，只需要一个简单的报告就能满足他。

明确仪表板最终用户

如果真的有必要制作仪表板，则需要明确最终仪表板的使用者是谁，考虑到用户体验，针对特定的使用者设计特定的仪表板。

明确衡量好坏的指标及其判断标准

仪表板大都是通过一些关键性的 KPI 指标来帮助我们回答某些业务问题，因此，需要针对问题，选定合适的数据和指标，并确定好评判标准来支撑我们的分析结论。

一个仪表板通常由多个部分构成，每个部分都用来回答某一个细化问题，它的一般布局如下：

列出所需的数据源

针对仪表板布局，获取相关数据。这里涉及到数据的访问权限和更新时间问题，因此，需要与相关人员提前沟通，方便制定仪表版后期的维护计划。

明确观察和筛选的维度

在制作仪表板之前，需要明确从哪些角度去观察分析相关指标。设计好筛选维度能帮助我们对指标进行交叉分析，多角度挖掘指标的影响因子。

确定需要钻取的详细信息

有些信息如果需要更细颗粒度的分析（如年、月、日），则需要制定好钻取的详细信息，方便用户单独查看和分析某一个具体的问题。

建立更新时间表

根据数据更新的频率，制定仪表板更新的时间表，保持信息的有效性和及时性。

Excel Dashboards 设计原则

Rule number 1: Keep it simple

Dashboard design expert Stephen Few has the mantra, “Simplify, Simplify, Simplify.”

只保留与主题强相关的数据信息

仪表板当中的每个组成部分都是为了解释某一个特定问题的，因此，每个组成部分也只需保留与解决该细化问题强相关的数据信息即可！

拒绝使用各种花里胡哨的格式

朴素才是真美！过多的格式和颜色会分散我们的注意力，而忽略了最有价值的信息！

这里有一些简单的建议，可以防止你过度夸张：避免使用颜色或背景填充来组织仪表板。

  1. 通常，颜色的使用应该是有节制的，只保留关键数据点的信息。例如，传统上，分配红、黄、绿来度量表示绩效水平。给仪表盘着色只会分散用户对你的信息的注意力。
  2. 不强调边框、背景和定义仪表板区域的其他元素。尝试使用组件之间的自然空白来划分仪表板。如果需要边框，请将其格式化为比数据更亮的颜色。浅灰色是典型的理想的边界。这样做的目的是在不分散信息注意力的情况下显示节显示出来。
  3. Excel 2013可以很容易地应用效果，让所有的东西看起来闪闪发亮，闪闪发亮，总体上很开心。虽然这些格式化功能是很好的营销工具，但它们对你或你的仪表盘没有任何帮助。避免应用花哨的效果，如渐变、模式填充、阴影、发光、柔边和其他格式。
  4. 不要试图用剪贴画或图片来增强您的仪表板。它们对进一步的数据表示没有任何帮助，而且通常看起来很俗气。

Skip the unnecessary chart junk（图表垃圾元素）

用于绘制图表的“墨水”更多应该用在数据上面，而不是其他元素，因此，要不不必要的图表元素都去掉，如：边界线、网格线、标签、背景、趋势线等等。（无助于核心元素的其他元素统统删掉！）

这里有一些方法来避免图表垃圾，并确保你的图表清楚地呈现你的数据。

  1. 移除网格线：网格线(垂直和水平)几乎总是不必要的。使用网格线的隐含原因是，它们有助于可视化地测量每个数据点所表示的值。然而，事实是，人们通常通过比较图表中其他数据点的位置来衡量数据点的价值。所以网格线成为次要的参考点，简单地占用墨水。
  2. 移除边框：你会发现，消除边框和框架会让你的图表看起来更清晰，并帮助你避免在一个仪表板上放置多个带边框的图表时出现令人眼花缭乱的线条。使用图表之间的空白代替边框。
  3. 跳过趋势线：趋势线很少提供不能通过已经绘制的数据或简单的标签获得的洞察。事实上，趋势线往往陈述明显的东西，有时会使读者误以为它们是另一个数据系列的一部分。当折线图本身就是某种趋势线时，为什么要在折线图上放一条趋势线呢？为什么要在柱状图上放置趋势线呢？代替趋势线，添加一个简单的标签，说明您试图对数据的总体趋势说些什么。
  4. 避免不必要的数据标签：没有说明您需要为您的图表中的每个值显示数据标签。可以绘制数据点而不显示其值。当你只显示与你的信息相关的数字时，你会发现你的图表更有影响力。
  5. 如果没有必要，就不要显示图例：在绘制一个数据系列时，不需要显示占用空间的图例。允许你的图表标题来识别你的图表所代表的数据。
  6. 删除任何不能增加价值的轴：X和y坐标轴的目的是帮助用户直观地测量和定位每个数据点所表示的值。但是，如果图表的性质和用途不需要一个特定的轴，那么删除它。再次强调，这里的目标不是砍断你的图表。目标是只包括那些直接有助于你的图表核心信息的图表元素。

保证一个仪表板一个页面

一个仪表板，使用一个页面，关注一个主题。如果信息较多，可以删除非必要元素，使用交互式的方式增加信息密度。

周围的颜色、边框、字体和其他格式会影响读者的浏览模式，弱化先前突出的区域。

Rule number 3: 统一格式化数字

在格式化仪表板和报表中的数字时，请记住以下几点：

  1. 使用逗号使数字更容易阅读。例如，显示2,345，而不是2345。
  2. 只有在需要那种精度的情况下才使用小数位。例如，在美元金额(如$123.45)中显示小数位很少有好处。同样，在百分比中，只使用有效表示数据所需的最小小数。例如，你可能会得到43%，而不是43.21%。
  3. 只有当你需要澄清你指的是货币价值时，才使用美元符号。如果你有一个包含所有收入值的图表或表格，并且有一个标签清楚地说明了这一点，你可以省去美元符号来节省空间和像素
  4. 将非常大的数字格式化为千位或百万位。例如，您可以将数字格式化为读取17M，而不是显示16,906,714。

Rule number 4: 有效地使用标题和标签

这是常识，但人们经常不能有效地在仪表板上标记项目。如果你的客户看着你的仪表盘问：“这告诉了我什么？”那么，你可能有标签问题。

这里有一些指导原则，可以有效地在仪表板和报告中进行标记：

始终在仪表板或报表上包含时间戳。这样，在每月或每周分期分发相同的仪表板或报告时，可以最大限度地减少混乱。

始终包含一些说明何时检索度量数据的文本。在许多情况下，数据的时间是分析度量的关键信息。

为每个组件使用描述性标题。这允许用户清楚地识别他们在看什么。一定要避免有许多首字母缩写和符号的晦涩的标题。

虽然这似乎违反直觉，但通过将标签格式化为比数据更亮的颜色来降低标签的重要性通常是一个好做法。浅色标签为用户提供他们需要的信息，而不会分散他们对所显示的信息的注意力。理想的标签颜色是在自然界中常见的颜色：软灰色、棕色、蓝色和绿色。

发布仪表板之前的必要检查

在正式发布仪表板之前，通常，我们需要问如下问题来进行检查：

我的仪表板是否展示了正确的信息？

再次核对核心目标，与需求方确认，查看仪表板是否符合正确的目标。

我的仪表板上所有的元素是否都有明确的目的导向？

精简元素，即使保留很多空白，也要删除非必要信息。

我的仪表板是否突出了关键信息？

可以参考布局的正确方式。

这个仪表板后期是否可以进行持续维护？

能否进行自动刷新和重复使用非常重要！

是否说明了仪表板的数据时效性？

说明源数据是哪个时间段的，可以保证数据的准确性和时效性。

是否有较为详细的使用文档说明？

一份较为详细的文档说明，可以帮助我们防止过段时间将仪表的制作和使用方式遗忘，导致效率极低的重复性劳动。

文档甚至不需要什么花哨的东西。一些简单的事情可以帮助记录您的仪表板。

  1. 向数据模型添加Model Map选项卡。Model Map选项卡是一个单独的表，您可以使用它总结数据模型中的键范围，以及每个范围如何与最终表示层中的报表组件交互。
  2. 尽量使用注释和标签。即使在您离开数据模型很长一段时间之后，一些解释性注释和标签仍然能够帮助阐明您的模型，这一点令人惊讶。

我的仪表板是否对用户友好？

  1. 直观：对于从未见过您的仪表板的人来说，您的仪表板应该是直观的。在某人身上测试一下，问问她这样做是否合理。如果你不得不开始解释仪表盘说了什么，那一定是出了问题。仪表盘是否需要更多的标签，更简单的图表，更好的布局，更多的数据，更少的数据？从几个用户那里获得反馈是个好主意。
  2. 易于导航：如果您的仪表板是动态的，允许与宏或透视表交互，那么您需要确保导航工作正常。用户必须点击几个地方才能得到他们的数据吗？钻取的次数合适吗？从一个视图切换到另一个视图需要太长时间吗？同样，您需要在多个用户上测试仪表板。一定要在其他电脑上测试交互式仪表盘功能。
  3. 正确打印：没有什么比打印仪表板却发现创建仪表板的人没有花时间确保正确打印更恼人的了。请确保您在Excel文件上设置了打印选项，以便正确打印仪表板。

我的仪表板的数据来源是否可靠？

阐明你的数据来源和真实性，增加你仪表板的可信度。增加业务经验，可以提高数字敏感度，避免一些计算失误。可以将仪表板给业务专家先检查一遍。
